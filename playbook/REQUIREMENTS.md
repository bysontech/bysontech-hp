# 要件定義

## 目的（1文）
- BysonTechの営業サイトとして、PoC開発サービスとPoCテンプレート販売サービスの内容・対象顧客・料金・進め方を明確に伝え、問い合わせや相談につなげる。

## MVPスコープ

### Must（必須）
- BysonTechの営業サイトとして公開できるトップページを作成する。
- 2つのサービスモデルを分かりやすく掲載する。
  - 小規模DX向けPoC開発
  - PoCテンプレート販売＋限定受注モデル
- 各サービスについて、少なくとも以下を掲載する。
  - 概要
  - 対象顧客
  - 提供内容
  - 価格帯
  - 提供範囲
  - 向いている人 / 向いていない人
- サイト全体の基本ページを用意する。
  - Top
  - Services
  - Pricing
  - FAQ
  - Contact
- 「短期・低コスト・テンプレート活用・保守なし / 限定対応」といった差別化ポイントを明示する。
- 営業サイトとして読みやすい情報設計にする。
  - ファーストビューで提供価値が分かる
  - 主要導線が分かりやすい
  - セクションごとの役割が明確
- 問い合わせ導線を設置する。
  - Contactページ
  - メール、外部フォーム、Upwork、LinkedInなどのいずれかに遷移可能
- 英語圏案件も意識した文脈を持たせる。
  - 海外小規模事業者向けサービスであることが伝わる構成
- Cloudflare Pages + Astro + Tailwindの静的サイトとして構築する。
- スマホ・PCの両方で大きく崩れないレスポンシブ対応を行う。
- 最低限のSEO設定を行う。
  - title
  - meta description
  - OGPの基本設定
- ブランドの信頼性を損なわないシンプルで軽量なデザインにする。
- 免責や注意事項を明記する。
  - PoC用途であること
  - 本番運用保証ではないこと
  - 保守・無制限修正は含まれないこと

### Should（できれば）
- 2つのサービスを「別事業」ではなく「1つのPoC支援ブランドの提供形態違い」として整理する。
- Template A / B / C / D など、今後のテンプレ展開を示せる構成にする。
- FAQで事前質問を減らせるようにする。
- 開発フローまたは導入フローを図やステップ形式で示す。
- 料金ページにプラン比較表を設ける。
- 英語ページ、または将来の英語対応を見越した構成にする。
- 実績・サンプル・ユースケースを今後追加しやすい構造にする。
- CTAを各ページ下部にも配置し、問い合わせ導線を強化する。
- 軽微なインタラクションを追加する。
  - FAQ開閉
  - セクション内アンカー
  - 比較カードの強調表示
- 利用規約、プライバシーポリシー、特定商取引法表記などを追加しやすい構成にする。

### Could（余裕があれば）
- ブログ / Insights / Notes のような発信用セクションを追加する。
- テンプレート一覧ページを作成し、各テンプレートの用途を簡潔に整理する。
- 「向いているプラン診断」のような軽い案内コンテンツを付ける。
- 実績カードやケーススタディのダミー枠を用意する。
- OGP画像やブランドロゴの簡易版を作成する。
- 将来的なSelf Build Kit販売に向けて、外部決済リンクを差し込める導線枠を用意する。
- 英語版LPを追加できるよう、コンテンツ分離を意識した実装にする。

## やらないこと
- 会員登録、ログイン、マイページなどのアプリ機能は作らない。
- 顧客管理、案件管理、進捗管理などの業務システムはこのサイトに載せない。
- 複雑な見積もりシミュレータや多段階フォームはMVPでは実装しない。
- ブログCMSや管理画面はMVPでは導入しない。
- 決済機能を自前実装しない。
- 保守受付システムやチャットサポート機能は作らない。
- フルオーダーメイド開発を前提とした訴求はしない。
- SEOを狙った大量コンテンツ制作はMVPでは行わない。
- アニメーション過多な演出や重いフロント実装は行わない。
- Next.jsのようなSSR前提の複雑な構成にはしない。

## 成功指標（仮）
- 初見の訪問者がトップページを見て、何を提供しているサイトか30秒以内に理解できる。
- 2つのサービスの違いがページ上で明確に区別できる。
- 問い合わせ導線が全ページから分かりやすく到達できる。
- LighthouseのPerformance / Accessibility / Best Practices / SEOで大きな問題が出ない。
- スマホ表示で主要セクションが読みづらくならない。
- 公開後、ポートフォリオや営業導線として使える最低限の完成度を満たす。
- 将来的なページ追加や英語展開を妨げない構造になっている。
