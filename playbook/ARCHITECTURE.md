# 基本設計（最小）

## 画面・機能一覧

### ページ構成

|ページ     |パス          |役割                 |主要セクション                                  |
|--------|------------|-------------------|-----------------------------------------|
|Top     |`/`         |ファーストビュー＋サービス概要＋CTA|ヒーロー、サービス概要カード×2、差別化ポイント、CTA             |
|Services|`/services/`|2つのサービスの詳細説明       |PoC開発詳細、テンプレート販売詳細、比較表、向いている人/向いていない人、CTA|
|Pricing |`/pricing/` |料金体系・プラン比較         |プラン比較表、含まれるもの/含まれないもの、免責事項、CTA           |
|FAQ     |`/faq/`     |よくある質問（開閉式）        |カテゴリ別Q&A（サービス全般、PoC開発、テンプレート、契約・支払い）、CTA |
|Contact |`/contact/` |問い合わせ導線            |連絡手段一覧（メール、外部フォーム、Upwork、LinkedIn）、簡易説明  |

### 共通コンポーネント

|コンポーネント    |役割                                               |
|-----------|-------------------------------------------------|
|Header     |ロゴ、ナビゲーション（5ページ）、モバイルハンバーガーメニュー                  |
|Footer     |コピーライト、ナビリンク、将来の法的ページリンク枠（利用規約・プライバシーポリシー等）      |
|CTA Section|各ページ下部に配置する共通の問い合わせ誘導ブロック                        |
|SEO Head   |title / meta description / OGP を各ページで設定するレイアウト共通部|

### 機能一覧

|機能        |実装方法                                             |
|----------|-------------------------------------------------|
|FAQ開閉     |HTML `<details>` / `<summary>` またはAstroアイランドで最小JS|
|モバイルメニュー  |Astroアイランド（最小JS）でハンバーガー開閉                        |
|セクション内アンカー|HTMLのid属性 + ナビリンク                                |
|外部リンク遷移   |Contact各手段は外部URLへ `target="_blank"`              |
|レスポンシブ    |Tailwindのブレークポイント（`sm:` / `md:` / `lg:`）         |

## データモデル（最小）

静的サイトのため、データベースは不使用。コンテンツは以下の形式で管理する。

### コンテンツデータ（TypeScriptオブジェクト）

```
src/
  content/
    services.ts      # サービス2種の詳細データ
    pricing.ts        # 料金プラン定義
    faq.ts            # Q&Aデータ配列
    contact.ts        # 連絡手段リスト
    site.ts           # サイト共通情報（サイト名、説明、OGPデフォルト値）
```

|データ     |主要フィールド                                                                              |
|--------|-------------------------------------------------------------------------------------|
|services|id, name, summary, targetCustomer, deliverables, priceRange, scope, fitFor, notFitFor|
|pricing |id, planName, price, features[], notIncluded[], disclaimer                           |
|faq     |id, category, question, answer                                                       |
|contact |id, method, label, url, description                                                  |
|site    |siteName, defaultDescription, ogImage, lang                                          |

**設計意図**: コンテンツをTSファイルに分離することで、将来の英語版対応（`content/en/` ディレクトリ追加）やCMS移行を容易にする。

## 非機能

|項目       |方針                                                                              |
|---------|--------------------------------------------------------------------------------|
|コスト上限    |月額 $0（Cloudflare Pages無料枠内。独自ドメインのみ実費）                                          |
|ログ方針     |Cloudflare Pagesの標準アクセスログのみ。アクセス解析はMVP後にCloudflare Web Analyticsを検討（無料・Cookie不要）|
|監視最低限    |Cloudflare Pagesのデプロイステータス確認のみ。外形監視はMVP後に検討                                     |
|データ保持方針  |ユーザーデータの取得・保持なし。フォーム送信は外部サービス側で管理                                               |
|パフォーマンス目標|Lighthouse全項目90以上を目指す。Astro静的ビルド＋最小JSで達成見込み                                     |
|可用性      |Cloudflare CDNに依存（実質的に高可用）                                                      |

## セキュリティ最低限

|項目     |方針                                                                  |
|-------|--------------------------------------------------------------------|
|秘密情報   |環境変数管理。MVPでは秘密情報はほぼ無し（外部サービスAPIキー等が将来発生した場合はCloudflare Pages環境変数を使用）|
|権限境界   |GitHubリポジトリへのpush権限 = デプロイ権限。Cloudflareアカウントアクセスは個人のみ               |
|外部公開範囲 |静的HTMLのみ公開。サーバーサイド処理なし。攻撃面は最小                                       |
|依存ライブラリ|npm auditでビルド時チェック。ランタイム依存はほぼゼロ                                     |
|メール保護  |メールアドレスの直書きを避け、`mailto:` リンクまたは外部フォーム経由に限定                          |
|HTTPヘッダ|Cloudflare Pagesデフォルト + `_headers` ファイルでCSP等の基本セキュリティヘッダを設定         |
